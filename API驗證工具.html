1 API URL 網址檢查

API 之 URL網址主要包括服務根網址(service root url, SRU)、資源路徑(resource path)及查詢選項(query options)。

  • 服務根網址:SRU 之內容包括scheme、host 及 port(通訊埠)等三項資訊。在本規範中，「scheme」一律採用 HTTP 或 HTTPS；「host」由開發單位設計，通常為主機名稱、單位名稱、單位性質、所在國家名稱縮寫等各類資訊之組合。必要時須指定特定的「port」(通訊埠)
  • 資源路徑:以階層式樹狀架構連結該 API 所提供之所有資源及其相關說明，各單位可依其服務內容設計(例如資料種類)，若有多重版本，版本資訊建議列於「path」之最前頭。OGC API 標準中保留部分關鍵字，用於指定特定用途之資訊(例如以「/confornamce」取得該 API 所符合標準之名稱)
  • 查詢選項:運用於查詢與取得符合條件之資料，常以參數方式表示。

    2 「Landing Page」檢查
  • API 須提供一個「landing page」，用以連結提供之資源及提供必要之說明，包括如 API 目的及可進行操作等。
  • 「landing page」可透過指定 SRU，並以 HTTP GET 之操作取得。
  • 該 API 所提供之相關資源可進一步結合「SRU」及「path」資訊後取得。
  • 「landing page」之回傳內容須包括以下資源之連結(links)，並可以 OpenAPI 之 schema 描述。
  • 「API definition」：主要為「service」之說明。
  • 「conformance declaration」：以「conformsTo」說明該 API 設定之符合類別名稱或 URI。
  • 當提供資料時，「landing page」須增加「collection」之連結，並提供包括如識別碼(例如可識別特定資料集之資訊)、提供 資源之坐標參考系統、名稱及描述(選擇性項目)、空間或時間範圍(選擇性項目)等。

    4 特定查詢功能支援

    API 必須支援以下查詢參數：

    • bbox：以坐標極值限制回傳資料之空間範圍，可為二維或三維。判斷以是否具有交集(intersect)為條件。
    • datetime：針對日期與時間限制回傳資料，可為時間點或時間段。判斷以是否具有交集(intersect)為條件。
    • limit：限制資料之回傳筆數。

    建議解決方案：

    • 實作 datetime 查詢參數功能
    • 支援 ISO 8601 時間格式
